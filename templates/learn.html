广度优先搜索 (BFS)

按层次遍历，保证最短路径

🎯 核心概念

广度优先搜索（Breadth-First Search）是一种图遍历算法，从起始节点开始，逐层向外扩展，先访问所有距离为1的节点，再访问距离为2的节点，以此类推。

📚 工作原理

使用队列（Queue）数据结构，遵循"先进先出"原则。每次从队列中取出一个节点，访问其所有未访问的邻居节点，将它们加入队列末尾。

⚡ 时间复杂度

O(V + E)，其中V是节点数，E是边数。BFS会访问每个节点和边各一次。

💡 适用场景

寻找无权图的最短路径、社交网络中找朋友、迷宫求解、Web爬虫等场景。

BFS特点：按层遍历（蓝色高亮为当前层），保证找到最短路径。从起点(0,0)开始，先访问第1层（右、下），再访问第2层。

📝 Python代码实现
def bfs(graph, start): visited = set() queue = [start] visited.add(start) while queue: node = queue.pop(0) print(node, end=' ') for neighbor in graph[node]: if neighbor not in visited: visited.add(neighbor) queue.append(neighbor) graph = { 'A': ['B', 'C'], 'B': ['A', 'D', 'E'], 'C': ['A', 'F'], 'D': ['B'], 'E': ['B', 'F'], 'F': ['C', 'E'] } bfs(graph, 'A') # 输出: A B C D E F

迷宫游戏

使用BFS算法寻找最短路径

💡 算法理解

在迷宫游戏中，BFS是最适合寻找最短路径的算法。它像水波纹一样向外扩散，第一次到达终点时走过的路径就是最短路径。这是因为BFS按距离递增的顺序访问节点，所以不会错过更短的路径。相比之下，DFS可能会陷入一条很长的死路，无法保证找到最短路径。

深度优先搜索 (DFS)

深入探索，回溯求解

🎯 核心概念

深度优先搜索（Depth-First Search）是一种图遍历算法，从起始节点开始，沿着一条路径尽可能深地探索，直到无法继续再回溯。

📚 工作原理

使用栈（Stack）数据结构，遵循"后进先出"原则。可以用递归或显式栈实现，每次访问一个节点后，立即探索其未访问的子节点。

⚡ 时间复杂度

O(V + E)，其中V是节点数，E是边数。与BFS相同，但空间复杂度可能更高（递归栈深度）。

💡 适用场景

拓扑排序、连通性检测、路径寻找、N皇后问题、数独求解等需要探索所有可能性的场景。

📝 Python代码实现
def dfs(graph, node, visited=None): if visited is None: visited = set() visited.add(node) print(node, end=' ') for neighbor in graph[node]: if neighbor not in visited: dfs(graph, neighbor, visited) graph = { 'A': ['B', 'C'], 'B': ['A', 'D', 'E'], 'C': ['A', 'F'], 'D': ['B'], 'E': ['B', 'F'], 'F': ['C', 'E'] } dfs(graph, 'A') # 输出: A B D E F C

N皇后问题

使用回溯法（DFS变体）放置皇后

💡 算法理解

N皇后问题是DFS回溯算法的经典应用。算法尝试在每行放置一个皇后，检查是否与已放置的皇后冲突（同列或对角线）。如果冲突，就回溯到上一行尝试其他位置。这种"尝试-失败-回溯"的模式正是DFS的精髓，能够系统地探索所有可能的解决方案。

A* 算法

启发式搜索，智能寻路

🎯 核心概念

A*是一种启发式搜索算法，结合了Dijkstra算法的准确性和贪心算法的效率，通过启发函数估计到达目标的代价。

📚 工作原理

使用优先级队列，每次选择f(n) = g(n) + h(n)最小的节点。g(n)是从起点到n的实际代价，h(n)是从n到终点的估计代价。

⚡ 启发函数

常用曼哈顿距离或欧几里得距离作为启发函数。h(n)必须是可采纳的（不超过实际距离），才能保证找到最优解。

💡 适用场景

路径规划、游戏AI寻路、八数码问题、地图导航等需要高效找到最优路径的场景。

📝 Python代码实现
import heapq def astar(start, goal, graph, heuristic): open_set = [(0, start)] came_from = {} g_score = {start: 0} while open_set: _, current = heapq.heappop(open_set) if current == goal: return reconstruct_path(came_from, current) for neighbor in graph[current]: tentative_g = g_score[current] + 1 if tentative_g < g_score.get(neighbor, float('inf')): came_from[neighbor] = current g_score[neighbor] = tentative_g f_score = tentative_g + heuristic(neighbor, goal) heapq.heappush(open_set, (f_score, neighbor)) return None def heuristic(a, b): return abs(a[0] - b[0]) + abs(a[1] - b[1]) # 曼哈顿距离

八数码问题

使用A*算法找到最优解

💡 算法理解

八数码问题是A*算法的经典应用场景。由于状态空间巨大（9!/2 = 181440种状态），BFS效率太低。A*使用曼哈顿距离作为启发函数，能够智能地引导搜索方向，大大减少搜索节点数，快速找到最优解。这展示了启发式搜索在复杂问题中的强大优势。

回溯法

试错探索，找到所有解

🎯 核心概念

回溯法是一种基于DFS的问题求解策略，通过尝试所有可能的选择，当发现当前选择无法得到有效解时，回溯到上一步尝试其他选择。

📚 工作原理

递归地构建解空间树，在每个节点选择一个分支深入探索，如果发现约束不满足则回溯，尝试其他分支。

⚡ 剪枝技巧

通过提前判断某些分支不可能产生有效解，避免不必要的搜索，显著提高效率。

💡 适用场景

组合优化问题、排列组合、N皇后问题、数独求解、子集和问题等需要枚举所有可能性的场景。

📝 Python代码实现
def solve_nqueen(n): result = [] board = [-1] * n def is_valid(row, col): for i in range(row): if board[i] == col or abs(board[i]-col) == abs(i-row): return False return True def backtrack(row): if row == n: result.append(board[:]) return for col in range(n): if is_valid(row, col): board[row] = col backtrack(row + 1) board[row] = -1 backtrack(0) return result solutions = solve_nqueen(4) print(f"4皇后问题共有{len(solutions)}种解法")

数独游戏

使用回溯法填充数字

💡 算法理解

数独游戏是回溯法的典型应用。算法尝试在每个空格填入1-9的数字，检查是否满足数独规则（同行、同列、同宫格不重复）。如果填入某个数字后无法继续，就回溯到上一个空格尝试其他数字。通过约束传播和剪枝技巧，可以大大提高求解效率。
